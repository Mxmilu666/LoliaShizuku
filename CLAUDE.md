# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

LoliaShizuku (「ロリア・雫」) is a third-party desktop client for Lolia FRP, built with [Wails v2](https://wails.io) (Go backend + Vue 3 frontend in a single native window). It handles OAuth login, a dashboard for user/traffic/tunnel data, and managing a local `frpc` runner process (install/update/start/stop with live logs).

## Commands

All commands run from the repository root unless noted.

```bash
wails dev          # Run full app in dev mode (auto-runs `bun install`, hot-reloads frontend + Go)
wails build        # Production build → build/bin/
```

Frontend-only (from `frontend/`):

```bash
bun install
bun run dev        # Vite dev server
bun run build      # vue-tsc typecheck implied via build; outputs to frontend/dist
```

There is **no test suite** and no separate lint step in this repo. Use `go vet ./...` and `go build ./...` to sanity-check Go changes; the frontend `bun run build` doubles as a typecheck.

### Prerequisites

- Go `>= 1.24`, Bun, and the Wails CLI (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`).
- `CGO_ENABLED=1` is required for builds (WebView bindings) — see `.github/workflows/release.yml`.

## Architecture

### Backend ↔ Frontend bridge (the core pattern)

The frontend never makes HTTP calls to the Lolia API directly. Instead, Go service structs are **bound** in `main.go` (`Bind: []interface{}{...}`), and Wails generates JS bindings under `frontend/wailsjs/go/`. Every exported method on a bound struct becomes callable from JS.

Two access styles appear in the frontend:
- Generated typed bindings: `import { GetDashboard } from '@/../wailsjs/go/services/CenterService'`.
- Runtime lookup with hand-written wrappers: `(window as any).go.services.CenterService.GetDashboard()` — see `frontend/src/services/center.ts` and `frontend/src/services/frpc.ts`, which wrap bindings, normalize errors (Chinese messages), and re-export typed interfaces.

**When you add or change an exported Go service method, the JS bindings in `frontend/wailsjs/go/` are regenerated by `wails dev`/`wails build`.** Don't hand-edit them; restart the Wails dev process to pick up signature changes.

Backend → frontend push uses Wails runtime events: `runtime.EventsEmit(ctx, "window_changed", ...)` in `system_service.go`, consumed via `EventsOn("window_changed", ...)` in `AppHeader.vue`. This is how the frameless custom title bar stays in sync.

### Bound Go services (`backend/services/`)

- `App` (`backend/app.go`) — config CRUD (`GetConfigJSON`/`UpdateConfig`/`ResetConfig`), holds the Wails `ctx`.
- `PreferencesService` — window size persistence.
- `TokenService` — `HasOAuthToken` and token presence checks (used by the router auth guard).
- `CenterService` — the main API client (dashboard, tunnels, traffic) **and** the local frpc runner lifecycle (`StartRunner`/`StopRunner`/`GetRunnerRuntimeStatus`). Runner state is mutex-guarded (`runnerMu`); logs are ring-buffered to `runnerLogMaxLines` (300). The runner is force-stopped in `OnBeforeClose`/`OnShutdown`.
- `FrpcService` — downloads/installs/updates/removes the `frpc` binary from GitHub releases, with configurable download mirrors.
- `systemService` (`services.System()`, singleton) — polls window state in a loop and emits `window_changed`.

### HTTP / API layer

- `backend/httpclient/client.go` — generic client that unwraps the Lolia API envelope (`{code, status, msg, data}`), injects the OAuth bearer token via a `GetAccessToken` callback, and invokes `OnUnauthorized` (which clears the stored token) on 401. Returns typed `*APIError`.
- `backend/api/center.go` — typed endpoint methods returning models from `backend/models/`.
- `backend/api/github_release.go` — GitHub release lookups for frpc.

### OAuth flow

Desktop login is **Authorization Code + PKCE** (fixed). Files: `oauth_flow.go`, `oauth_token.go`, `token_service.go`, `token_store.go`. Tokens are stored in the OS **keyring** (`zalando/go-keyring`) under service `LoliaShizuku`, key `oauth_token`. Default callback listens on `http://localhost:1145`. `loadOrRefreshOAuthToken` handles refresh; on failure the token is cleared and the router guard (`frontend/src/router/index.ts` `beforeEach`) redirects to `/oauth`.

### Configuration (two distinct stores — don't confuse them)

1. **App config** (`backend/config/config.go`) — JSON `Config` struct (theme, window, advanced) stored on disk via the `Manager`. Loaded *before* the window opens in `main.go` to restore window size.
2. **frpc state** — under `os.UserConfigDir()/LoliaShizuku/userdata/frpc/`: `bin/` (binary), `downloads/` (cache), `installed.json` (install state), `settings.json` (mirror settings).

Many behaviors are overridable by env vars (API base URL, OAuth client/URLs, frpc repo, HTTP UA) — see the table in `README.md`. Defaults point at `api.lolia.link` / `dash.lolia.link` and the `Lolia-FRP/lolia-frp` GitHub repo.

### Frontend (`frontend/src/`)

Vue 3 + TypeScript + Vuetify 3 (Material Design 3 blueprint) + Pinia + vue-router. Vite alias `@` → `frontend/src`. Theme (`plugins/theme.ts`, light/dark/system) is set up in `main.ts` and synced with system preference via `matchMedia`; the choice is persisted in `localStorage` (`lolia.theme`), separate from the Go-side config.

- `pages/` — one folder per route: `home`, `oauth`, `tunnels`, `runner`, `settings`. Routes are lazy-loaded in `router/index.ts`, guarded by OAuth token presence.
- `services/` — typed wrappers over Wails bindings (`center.ts`, `frpc.ts`).
- `stores/` — Pinia (`frpcInstall.ts` for install progress, `globalLoading.ts`).
- The window is **frameless** (`Frameless: true` in `main.go`); `AppHeader.vue` is the custom draggable title bar.

## Conventions

- Code comments and user-facing strings are frequently in **Chinese**; match the surrounding style.
- Platform-specific process handling lives in `process_attr_windows.go` / `process_attr_other.go` (build-tag split) — keep both in sync when changing runner process spawning.
- Version is centralized in `backend/version/version.go` (used for the HTTP User-Agent and update checks).
